## What you'll learn

- **Postgres:** How to enable logical replication on RDS Postgres (`rds_replication` role); the difference between a full-load + CDC task (Postgres target) and a CDC-only task (S3 target)
- **Oracle:** Why ARCHIVELOG mode on RDS Oracle is controlled by backup retention, not a direct SQL command; why RDS blocks `ALTER DATABASE` and requires the `rdsadmin` package for supplemental logging instead; why Oracle table mappings must be schema-scoped (a wildcard schema pulls in internal schemas like `GSMADMIN_INTERNAL`); the exact S3 IAM permissions (`DeleteObject`, `PutObjectTagging`) that are easy to miss and cause a silent-looking table suspension
- **Both:** How DMS captures row-level changes (INSERT / UPDATE / DELETE) and streams them into a data lake
